=== ТЕОРИЯ К ЛАБОРАТОРНОЙ РАБОТЕ 3 ===

1. ВИРТУАЛЬНАЯ ПАМЯТЬ:
- Каждый процесс "видит" своё линейное адресное пространство (виртуальное).
- ОС отображает (мапит) виртуальные адреса в физические через таблицы страниц.
- Процесс не знает, где в RAM его данные — этим управляет ядро.

2. ТАБЛИЦА СТРАНИЦ:
- Структура, где каждая запись связывает виртуальную страницу с физической.
- Page hit — страница найдена в RAM → быстрый доступ.
- Page fault — страницы нет в RAM → ОС загружает её с диска (swap) → медленно.

3. УСТРОЙСТВО ПАМЯТИ ПРОЦЕССА:
- .text — код (только чтение)
- .data — инициализированные глобальные переменные
- .bss — неинициализированные глобальные переменные
- Куча (heap) — динамическая память (malloc, mmap)
- Стек (stack) — локальные переменные, адреса возврата
- Общая память (shared memory) — мапится в кучу через mmap

4. СИСТЕМНЫЕ ВЫЗОВЫ:
- shm_open(name, flags, mode) — создаёт/открывает именованный объект в разделяемой памяти (аналог open() для файлов).
- ftruncate(fd, size) — задаёт размер сегмента.
- mmap(addr, length, prot, flags, fd, offset) — отображает файл или сегмент памяти в виртуальное адресное пространство процесса.
- munmap(addr, length) — отключает отображение.
- shm_unlink(name) — удаляет именованный объект (аналог unlink для файлов).

5. СУТЬ РАБОТЫ. ЧЕМ ОТЛИЧАЕТСЯ ОТ ЛР1:
- В ЛР1 обмен данными — через каналы (pipe): односторонний, FIFO, данные копируются через ядро.
- В ЛР3 обмен — через общую память (shared memory): оба процесса читают/пишут в один участок памяти, данные не копируются.
- Shared memory быстрее, но требует синхронизации (в отличие от pipe).
- Используются mmap/shm_open вместо pipe/dup2.
- Основная цель — показать альтернативный, более эффективный способ IPC (Inter-Process Communication).
