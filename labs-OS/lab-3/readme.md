# Лабораторная работа №3: Взаимодействие процессов через общую память

## Условие варианта (на примере 1-го варианта)

Родительский процесс создаёт **сегмент общей памяти** и запускает дочерний процесс.  
Пользователь вводит команды вида: `число число число`  
(произвольное количество чисел, тип `int`).  
Родитель записывает команду в **общую память**, дочерний процесс — читает оттуда, считает сумму, записывает результат в файл.  
Родитель и дочерний — разные программы.

Вариант 1: пользователь вводит строки чисел → дочерний считает сумму → пишет в файл.

## Требования:

- Использовать **shared memory** (через `shm_open`, `mmap`, `munmap`, `shm_unlink`)
- Только системные вызовы: `fork`, `shm_open`, `mmap`, `read`, `write`, `open`, `close`, `execve`, `_exit`, `wait`, `dup2`
- Запрещено: `stdio.h`, `stdlib.h`, `string.h`, `pthread`, `semaphore`, `msgqueue`

## Требования к среде:

- Linux или WSL
- gcc
- POSIX-совместимость

## Как собрать

'''bash
gcc -o parent parent.c -lrt
gcc -o child child.c -lrt
./parent

Enter output filename: result.txt
Enter lines with integers (empty line to quit):
> 10 20 30
> -5 100 3
> 

Parent finished.
'''
result.txt:
60
