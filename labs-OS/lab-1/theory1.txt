Ядро — центр ОС. Управляет процессами, памятью, устройствами. Только через него — доступ к железу.

Каналы (pipe) — односторонняя труба между процессами. pipe[0] — читать, pipe[1] — писать. FIFO. В памяти. Уничтожаются при завершении.

Память ПК (по скорости):
- Регистры (самые быстрые, в CPU)
- Кэш (L1/L2/L3, рядом с CPU)
- RAM (оперативка, временная)
- SSD/HDD (долгосрочная, медленная)

Целочисленные типы (C, 64-bit):
char   — 1 байт  — -128..127
short  — 2 байта — -32768..32767
int    — 4 байта — -2.1e9..2.1e9
long   — 8 байт  — -9.2e18..9.2e18
long long — 8 байт — то же, что long

Файловые дескрипторы:
0 — stdin
1 — stdout
2 — stderr
3+ — файлы, каналы

=== ЛАБОРАТОРНАЯ 1 — ОТВЕТЫ ===

1. СИСТЕМНЫЕ ВЫЗОВЫ:
- Это — интерфейс между программой и ядром ОС.
- Отличие от обычных функций:
  * Обычные: выполняются в пользовательском режиме (внутри программы).
  * Системные: вызывают код ядра (привилегированный режим).
  * Пример: printf() — обычная (из libc), write() — системный вызов.

2. СИСТЕМНЫЕ ВЫЗОВЫ ИЗ ЛАБЫ:
- fork() — создаёт копию процесса (родитель + ребёнок).
- pipe() — создаёт канал (два fd: 0-чтение, 1-запись).
- dup2(old, new) — копирует дескриптор (например, stdin = канал).
- execve() — заменяет текущий процесс новой программой.
- read/write — чтение/запись через дескриптор.
- open/close — открыть/закрыть файл.
- wait() — ждёт завершения дочернего процесса.
- _exit() — завершает процесс (без буферов, чисто).

3. УСТРОЙСТВО ПАМЯТИ ПРОЦЕССА:
- .text — машинный код (только чтение).
- .data — инициализированные глобальные переменные.
- .bss — неинициализированные глобальные переменные.
- Куча (heap) — динамическая память (malloc).
- Стек (stack) — локальные переменные, адреса возврата.

4. GIT:
- clone — скачать репозиторий.
- add — добавить файлы в индекс.
- commit -m "..." — зафиксировать изменения.
- push/pull — отправить/забрать изменения.
- status — посмотреть изменения.
- checkout -b имя — создать ветку.

5. ПО СИ:
- exit() vs _exit() — exit() вызывает буферы, _exit() — нет.
- fork() возвращает 0 в ребёнке, PID — в родителе.
- Файловый дескриптор — целое число, указывает на открытый файл.
- execve() не возвращается, если успешен — заменяет процесс.
